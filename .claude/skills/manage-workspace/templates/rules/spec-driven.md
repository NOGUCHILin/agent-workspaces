# 仕様駆動開発ルール

## 原則

1. **仕様優先**: 実装前に必ず仕様を確認・作成する
2. **調査記録**: 調べた内容はresearch/に記録する
3. **変更追跡**: 仕様変更時は上流ドキュメントも更新する

## ワークフロー

1. 新機能開始時: `docs/specs/{feature}/` を作成
2. 要件定義 → 設計 → タスク の順で作成
3. 調査が必要な場合は `research/` に記録
4. 実装中に仕様変更があれば上流を更新

## ディレクトリ構造

```
docs/specs/{feature}/
├── 01-requirements.md  # 何を作るか
├── 02-design.md        # どう作るか
├── 03-tasks.md         # 作業項目
└── research/           # 調査結果
    └── {topic}.md
```

## テンプレート

新規作成時は `docs/_templates/` のテンプレートを使用すること
