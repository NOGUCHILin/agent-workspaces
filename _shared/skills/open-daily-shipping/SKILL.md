# 発送業務ワークスペース準備スキル

発送業務に必要な3つのサイトを開き、適切なフィルターを設定するスキル。

## 前提条件

- Playwright MCPが有効であること
- 各サイトへのログイン情報がブラウザに保存されていること

## ワークフロー概要

1. ブラウザを開く（about:blank）
2. Notionを開く（タブ0）
3. Back Marketを開く（タブ1）
4. Kintoneを開く（タブ2）
5. ユーザーが各サイトにログイン
6. Back Marketで「発送予定」フィルターを適用
7. Notionで本日の日付フィルターを適用

## 詳細手順

### 1. ブラウザを開く

```
about:blank
```

### 2. Notionを開く（タブ0）

```
URL: https://www.notion.so/japanconsulting/ebaadbf255204fddb6098081a1715f9f?v=6bbe9978896c44c7a04ad8476491231c
```

発送予定リスト。ログイン画面が表示されたら、ユーザーにログインを依頼。

### 3. Back Marketを開く（タブ1）

新しいタブを作成して開く。

```
URL: https://www.backmarket.co.jp/bo-seller/orders/all?page=1&pageSize=100
```

Seller Portal注文一覧。ログイン画面が表示されたら、ユーザーにログインを依頼。

### 4. Kintoneを開く（タブ2）

新しいタブを作成して開く。

```
URL: https://japanconsulting.cybozu.com/k/11/
```

買取販売リスト。ログイン画面が表示されたら、ユーザーにログインを依頼。

### 5. ログイン待機

ユーザーが各サイトにログインするのを待つ。「ログインしました」等の合図を待つ。

### 6. Back Marketフィルター設定

**重要: 日付は指定しない**

1. タブ1（Back Market）を選択
2. 「注文ステータス」ドロップダウンで「発送予定」を選択
3. 開始日・終了日は空欄のまま
4. 「フィルターを適用する」をクリック

### 7. Notionフィルター設定

1. タブ0（Notion）を選択
2. 「フィルター」ボタンをクリック
3. 既存の日付フィルターをクリック
4. カレンダーで「Go to next month」等を使い、本日の日付に移動
5. 本日の日付をクリック
6. Escapeキーでダイアログを閉じる

## フィルター設定まとめ

| サイト | フィルター | 日付指定 |
|--------|-----------|---------|
| Back Market | 発送予定 | なし |
| Notion | 作成日時 | 本日 |
| Kintone | なし | - |

## 最終状態

- タブ0: Notion発送予定（本日フィルター適用）
- タブ1: Back Market注文一覧（発送予定フィルター適用）
- タブ2: Kintone買取販売リスト

## 注意事項

- Back Marketのフィルターでは日付を指定しない（過去の発送予定も含める）
- Notionのフィルターは本日の日付で絞り込む
- ログインはユーザーが手動で行う
- ブラウザがゲストモードの場合、パスワード自動入力が効かない可能性あり
