# IMEI入力スキル

売れて発送した端末のIMEIをBackMarketの注文に記録するスキル。

## 前提条件

- Playwright MCPが有効であること
- Notion、BackMarketへのログイン情報がブラウザに保存されていること

## ワークフロー概要

1. Notionの発送予定ページを開く（本日の日付でフィルター）
2. 別タブでBackMarketを開く（フィルターなし）
3. Notionで発送した顧客名を確認
4. BackMarketでその顧客名で検索
5. 該当の注文にIMEIを入力

## 詳細手順

### 1. Notionを開く

```
URL: https://www.notion.so/japanconsulting/ebaadbf255204fddb6098081a1715f9f?v=6bbe9978896c44c7a04ad8476491231c
```

発送予定リスト。**必ず本日の日付でフィルターする**。

フィルター手順:
1. 「作成日時: X月X日」フィルターボタンをクリック
2. カレンダーで本日の日付を選択
3. Escキーでダイアログを閉じる

### 2. BackMarketを開く（別タブ）

新しいタブを作成して開く。

```
URL: https://www.backmarket.co.jp/bo-seller/orders/all?page=1&pageSize=100
```

**重要: フィルターは適用しない**（ステータス「すべて」のまま）

理由: 発送済みの注文は件数が多すぎるため、フィルターではなく顧客名で検索する方が効率的。

### 3. IMEI入力作業

1. **Notionで確認**: 発送予定リストから発送済みの顧客を確認
   - 「備考 1」列にIMEIが未入力のものを探す

2. **BackMarketで検索**:
   - 「表示するフィルターを増やす」ボタンをクリック
   - 「お客様の名」フィールドに名前だけを入力（姓は不要、効率が良いため）
   - 「フィルターを適用する」をクリック

3. **注文を開く**: 該当の注文番号をクリック

4. **IMEIを入力**: 注文詳細画面でIMEIを入力・保存

5. **繰り返し**: 次の顧客へ

## 最終状態

- タブ0: Notion発送予定（本日フィルター適用）
- タブ1: BackMarket注文一覧（フィルターなし、検索用）

## 注意事項

- Notionを開く際は**必ず本日の日付でフィルター**する
- BackMarketはフィルターなしで開く（顧客名で検索）
- 「発送済み」フィルターは使わない（件数が多すぎるため）
