# kintone進捗一括変更スキル

Notion発送予定リストのバックマーケット注文について、kintoneの進捗を「出品完了」から「販売完了」に変更し、販売価格と販売日を入力する。

## 前提条件

- Playwright MCPが有効であること
- Notion、Back Market、kintoneにログイン済みであること
- Notion発送予定が今日の日付でフィルターされていること

## 手順

### 1. 対象レコードの確認

Notion発送予定から「バックマーケット」の注文を確認し、以下の情報を取得：
- 注文者名
- レコード番号（SKU）

### 2. kintoneで対象レコードを一括検索

**最初にkintoneで全対象レコードを一括検索して開いておく。**

一括検索URL形式：
```
https://japanconsulting.cybozu.com/k/11/?q=レコード番号 in ("337736", "337737", "336528")
```

例：レコード番号が 337736, 337737, 336528 の場合
```
https://japanconsulting.cybozu.com/k/11/?q=レコード番号 in ("337736", "337737", "336528")
```

これで対象レコードが一覧表示され、1件ずつ開くより効率的。

### 3. Back Marketで注文を検索

Back Market販売者管理ページ（https://www.backmarket.co.jp/bo-seller/orders/all）で検索。

#### 基本検索（SKUで検索）

1. 終了日を今日の日付に設定
2. SKU欄にレコード番号を入力
3. 「フィルターを適用する」をクリック

#### SKUで見つからない場合の対処法

SKU検索で「この絞り込み条件で見つかった注文はありません」と表示された場合：

1. **SKU欄をクリア**（×ボタンをクリック）
2. **「お客様の名」欄に名前（名字ではなく下の名前）を入力**
   - 例：「鈴木 悠矢」→「悠矢」で検索
   - 例：「森本 エリカ」→「エリカ」で検索
3. 「フィルターを適用する」をクリック

**注意**: 同じ名前で複数の注文がある場合があるため、必ず次のステップで照合を行う。

### 4. 顧客名の照合（重要）

**Back Marketの検索結果とNotionの発送予定の注文者名が一致していることを必ず確認する。**

確認ポイント：
- Back Marketの「お客様」列の名前
- Notionの「注文者」列の名前

**同じSKUで別の注文がある場合があるため、名前の一致確認は必須。**

一致していない場合：
- 検索条件を変更して再検索
- 終了日の範囲を広げる
- 姓と名を入れ替えて検索

### 5. 販売価格の取得

Back Marketの検索結果から「合計価格（税込）」を確認。
例：¥36,758

### 6. kintoneレコードの更新

kintoneの一括検索結果から対象レコードをクリックして開く。

1. 検索結果一覧から対象レコードをクリック

2. 「レコードを編集する」をクリック

3. 「販売」セクションを開く

4. 以下を入力：
   - **最終販売価格（税込）**: Back Marketの合計価格（数字のみ、例：36758）
   - **販売日**: 今日の日付（YYYY-MM-DD形式）

5. **進捗**を「販売完了」に変更

6. 「保存」をクリック

7. ブラウザの「戻る」で検索結果一覧に戻り、次のレコードを処理

## SKU末尾に文字がつくケース

Back MarketのSKUには末尾に「s」などの文字がつく場合がある。

例：
- kintoneレコード番号: 336286
- Back Market SKU: 336286s

この場合、SKU検索では見つからないため、名前検索を使用する。

## チェックリスト

各レコード処理時に確認：

- [ ] Back Marketの顧客名がNotionの注文者名と一致している
- [ ] 販売価格が正しく入力されている
- [ ] 販売日が今日の日付になっている
- [ ] 進捗が「販売完了」に変更されている
- [ ] レコードが保存されている

## 7. Notionで進捗変更タグを追加

kintoneの更新が完了したら、Notionの発送予定で対象レコードの「送り状」列に「進捗変更」タグを追加する。

### 手順（notion-tag-copyスキル使用）

1. **最初のセルにタグを追加**
   - バックマーケット最初のレコードの「送り状」セルをクリック
   - 「進捗変更」タグを選択
   - Escキーでダイアログを閉じる

2. **コピー**
   - Ctrl+C

3. **下のセルにペーストを繰り返す**
   - ↓（下矢印キー）
   - Ctrl+V
   - 残りのセルに対して繰り返す

### ポイント

- Escでダイアログを閉じてからコピーすること
- ペースト後は自動的に次のセルに進める状態になる

## 使用タイミング

- 「進捗変更して」
- 「販売完了にして」
- 「kintone更新して」
- 発送作業完了後
