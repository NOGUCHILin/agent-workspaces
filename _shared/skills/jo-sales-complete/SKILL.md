# 徐さん販売完了スキル

徐さんへiPhoneを販売した後、kintoneレコードを更新するスキル。

## 前提条件

- Playwright MCPが有効であること
- 対象レコードが既にkintoneに存在すること

## ログインが必要な場合

ログイン画面が表示されたら `kintone-login` スキルを使用してログイン。

## 必要な情報

ユーザーから以下を確認：
- レコード番号
- 最終販売価格（税込）（デフォルト: 10000円）
- 販売日（デフォルト: 当日）

## 手順

### 1. レコードを開く

```
URL: https://japanconsulting.cybozu.com/k/11/show#record={レコード番号}
```

### 2. 編集モードに入る

「レコードを編集する」をクリック。

### 3. 販売セクションを開く

「販売」ボタンをクリックしてセクションを展開。

### 4. 販売情報を入力

| フィールド | 値 |
|-----------|-----|
| 最終販売価格（税込） | 10000（または指定額） |
| 販売日 | 当日の日付（YYYY-MM-DD形式） |
| 販売経路 | 徐さん |

### 5. 進捗を変更

「進捗」ドロップダウンを開き、「販売完了」を選択。

### 6. 保存

「保存」ボタンをクリック。

## 注意事項

- 進捗を「販売完了」に変更する際、買取価格が未入力だとエラーになる
- その場合は「最終買取価格」を入力してから進捗を変更
- 利益がマイナスの場合、確認ダイアログが表示される → 承認して続行

## 複数レコードの場合

複数レコードを更新する場合は、各レコードに対して手順1〜6を繰り返す。

## 用途

- 徐さんへの販売完了処理
- 進捗ステータスの一括変更
