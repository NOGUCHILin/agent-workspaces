# 内部SEOフレームワーク

## 概要

内部SEOは、サイト内部の構造・リンク・メタデータを最適化し、検索エンジンとユーザーの両方にとって分かりやすいサイトを構築する施策。

## フレームワーク一覧

### 1. 内部リンク戦略フレームワーク

Googleが「SEOにとって超重要」と明言した要素。

**リンクモデル**:
| モデル | 説明 | 適用場面 |
|--------|------|----------|
| Hub-and-Spoke | ピラーページから放射状にリンク | トピッククラスター |
| Silo | テーマ別にグループ化 | 大規模サイト |
| Contextual | 本文中の文脈に沿ったリンク | 自然なリンク構築 |

**ルール**:
- 重要ページは3クリック以内でアクセス可能に
- 1500字あたり5本程度の内部リンク
- リンクは均等に分散配置
- フッターリンクの乱用は避ける

**ワークフロー**:
```
1. サイト構造を可視化
2. 孤立ページを特定
3. ピラーページを設定
4. クラスター間のリンクを設計
5. 本文中に文脈リンクを追加
6. パンくずリストを実装
7. 定期的にリンク切れチェック
```

### 2. サイト構造（IA）フレームワーク

情報アーキテクチャの設計。

**階層構造**:
```
ホーム（深度0）
├── カテゴリ1（深度1）
│   ├── サブカテゴリ1-1（深度2）
│   │   └── 記事（深度3）
│   └── サブカテゴリ1-2（深度2）
└── カテゴリ2（深度1）
```

**ルール**:
- 最大深度は3-4階層
- URL構造は階層を反映
- パンくずリストで現在地を明示

**ワークフロー**:
```
1. コンテンツをカテゴリ分類
2. 階層構造を設計（3-4階層以内）
3. URL設計（階層を反映）
4. ナビゲーション設計
5. パンくずリスト実装
6. サイトマップ作成
```

### 3. URL設計フレームワーク

SEOに最適化されたURL構造。

**ルール**:
- 短く簡潔に（50-60文字以内）
- キーワードを含める
- 小文字のみ使用
- ハイフンで区切る
- パラメータは最小限に

**良い例**:
```
/iphone-15-kaitori-souba/
/stores/akihabara/
```

**悪い例**:
```
/page?id=12345&cat=3
/iPhone_15_Buy_Back_Price_Guide_2025/
```

### 4. Canonical設計フレームワーク

重複コンテンツの正規化。

**実装ルール**:
- 各ページに1つのcanonical
- 自己参照canonicalを設定
- httpsを優先
- www有無を統一
- パラメータ付きURLは正規URLにcanonical

**ワークフロー**:
```
1. 重複コンテンツを洗い出し
2. 正規URLを決定
3. canonical tagを実装
4. Search Consoleで確認
5. 定期的に監視
```

### 5. メタタグ最適化フレームワーク

各ページのメタ情報最適化。

| タグ | 最適な長さ | ルール |
|------|-----------|--------|
| title | 60文字以内 | キーワード前方配置、ブランド名末尾 |
| description | 120-160文字 | CTA含む、ユニーク |
| h1 | 1ページ1つ | titleと類似、キーワード含む |

**ワークフロー**:
```
1. キーワードを決定
2. titleを作成（60文字以内）
3. descriptionを作成（120-160文字）
4. h1を設定（1つのみ）
5. h2-h3の階層を設計
6. 重複チェック
```

## チェックリスト

### 内部リンク
- [ ] 孤立ページがない
- [ ] 重要ページは3クリック以内
- [ ] パンくずリスト実装済み
- [ ] リンク切れがない

### サイト構造
- [ ] 階層は3-4以内
- [ ] URLが階層を反映
- [ ] ナビゲーションが明確

### メタタグ
- [ ] titleが60文字以内
- [ ] descriptionが120-160文字
- [ ] h1が1ページ1つ
- [ ] canonical設定済み
