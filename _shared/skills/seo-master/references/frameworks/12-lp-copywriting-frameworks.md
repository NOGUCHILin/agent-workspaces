# LPコピーライティングフレームワーク集

13種類のフレームワークを用途別に整理。適切なフレームワークを選択することでCV率向上。

## フレームワーク選択ガイド

| 商材タイプ | 推奨フレームワーク |
|-----------|-------------------|
| 悩み解決型（買取、修理） | PASBECONA、新PASONA |
| 理想実現型（美容、転職） | AIDCA、QUEST |
| 商品訴求型（EC、物販） | BEAF、12スキームLP |
| 簡潔訴求型（BtoB、SaaS） | CREMA、バズ部型 |

---

## 1. PASBECONA（パスビーコーナ）

**用途**: 悩み解決型商材の最強フレームワーク。神田昌典氏のPASONA発展形。

| ステップ | 要素 | 実装例（iPhone買取） |
|---------|------|---------------------|
| P | Problem（問題） | 「壊れたiPhone、どうすればいい？」 |
| A | Affinity（親近感） | 「修理費用が高くて悩んでいませんか？」 |
| S | Solution（解決策） | 「そのまま売れば現金に変わります」 |
| B | Benefit（利得） | 「修理費より高く売れることも」 |
| E | Evidence（証拠） | 買取事例、口コミ、実績数 |
| C | Contents（内容） | サービス詳細、3ステップ |
| O | Offer（提案） | 「LINE査定なら10秒で価格確認」 |
| N | Narrow（適合） | 「他社で断られた端末もOK」 |
| A | Action（行動） | CTAボタン |

---

## 2. QUEST（クエスト）

**用途**: 長文LP、セールスレター。ターゲットを絞り込んで教育する。

| ステップ | 要素 | 実装例 |
|---------|------|--------|
| Q | Qualify（限定） | 「画面が割れたiPhoneをお持ちの方へ」 |
| U | Understand（理解） | 「修理か売却か迷いますよね」 |
| E | Educate（教育） | 「実は修理より売却が得なケースが多い」 |
| S | Stimulate（刺激） | 「最大90%の価格で買取可能」 |
| T | Transition（変換） | 「今すぐLINEで査定」 |

---

## 3. AIDCA（アイドカ）

**用途**: 理想実現型商材。Conviction（確信）で不安を払拭。

| ステップ | 要素 | 実装例 |
|---------|------|--------|
| A | Attention（注意） | インパクトあるヘッドライン |
| I | Interest（興味） | メリット、差別化ポイント |
| D | Desire（欲求） | 理想の未来を描写 |
| C | Conviction（確信） | 口コミ、保証、実績で不安解消 |
| A | Action（行動） | CTA |

---

## 4. CREMA（クレマ）

**用途**: 簡潔なLP、BtoB。結論ファーストで忙しいユーザー向け。

| ステップ | 要素 | 実装例 |
|---------|------|--------|
| C | Conclusion（結論） | 「壊れたiPhoneも高価買取」 |
| R | Reason（理由） | 「パーツ再利用で価値がある」 |
| E | Evidence（証拠） | 買取事例、実績 |
| M | Method（手段） | 「LINE査定→集荷→即日振込」 |
| A | Action（行動） | CTA |

---

## 5. BEAF（ビーフ）

**用途**: 商品訴求型。ECサイト、物販LP。

| ステップ | 要素 | 実装例 |
|---------|------|--------|
| B | Benefit（利益） | 「最短即日で現金化」 |
| E | Evidence（証拠） | 買取実績15万人+ |
| A | Advantage（優位性） | 「他社で断られた端末もOK」 |
| F | Feature（特徴） | LINE査定、無料集荷、即日振込 |

---

## 6. 新PASONA（パソナ）

**用途**: 悩み解決型の基本形。PASBECONAの簡易版。

| ステップ | 要素 |
|---------|------|
| P | Problem（問題提起） |
| A | Affinity（親近感・共感） |
| S | Solution（解決策） |
| O | Offer（提案） |
| N | Narrowing（絞り込み） |
| A | Action（行動） |

---

## 7. 12スキームLP（ネットショップ総研）

**用途**: EC、物販。日本人に親しみやすい構成。

```
1. 注意喚起（キャッチ）← サビを最初に
2. 興味関心
3. 注意喚起（欲求）← サビ繰り返し
4. 差別化
5. ストーリー
6. 注意喚起（欲求）← サビ繰り返し
7. 購入想起
8. 論拠
9. 安心/安全
10. ベネフィット
11. 限定
12. クロージング
```

**ポイント**: JPOPの楽曲構造をモデルに、キャッチを繰り返す。

---

## 8. バズ部型

**用途**: シンプルで効果的な4要素構成。

| 要素 | 説明 |
|------|------|
| 結果 | 何が得られるか |
| 実証 | なぜその結果が得られるか |
| 信頼 | なぜあなたを信じるべきか |
| 安心 | リスクはないか |

---

## 9. FREE WEB HOPE型

**用途**: ストーリーテリング重視のLP。

```
1. 誘起（興味を引く）
2. 結果（得られるもの）
3. 証拠（実績、データ）
4. 共鳴（共感ストーリー）
5. 信頼（権威性）
6. ストーリー（開発秘話、想い）
7. クロージング
8. P.S.（追伸で念押し）
```

---

## 10. ロバート・コリアーの型

**用途**: クラシックなセールスレター構成。長文LP。

| ステップ | 要素 |
|---------|------|
| Opening | 注意を引く |
| Description | 描写・説明 |
| Motive | 動機・理由 |
| Proof | 証拠・保証 |
| Snapper | 決め手・不利益 |
| Close | クロージング |

---

## フレームワーク適用チェックリスト

### 選択時の判断基準

1. **悩みの深さ**
   - 深い悩み → PASBECONA、QUEST
   - 軽い悩み → CREMA、BEAF

2. **商材タイプ**
   - 悩み解決 → PASBECONA
   - 理想実現 → AIDCA
   - 商品訴求 → BEAF、12スキーム

3. **LPの長さ**
   - 長文 → QUEST、ロバート・コリアー
   - 中文 → PASBECONA、12スキーム
   - 短文 → CREMA、バズ部型

4. **ターゲット**
   - BtoC → 12スキーム、PASBECONA
   - BtoB → CREMA、バズ部型

## 実装例：壊れたiPhone買取LP

現在のScrollLP.tsxをPASBECONA観点で分析：

| 要素 | 現状 | 評価 |
|------|------|------|
| P: Problem | DiagnosticHero「壊れたiPhone」 | ○ |
| A: Affinity | SymptomSelectSection | ○ |
| S: Solution | RepairVsBuybackSection | ○ |
| B: Benefit | 「修理より買取が得」 | ○ |
| E: Evidence | BuybackCasesSection、口コミ | ○ |
| C: Contents | StepGuideSection | ○ |
| O: Offer | CTA（LINE査定） | △要強化 |
| N: Narrow | TradeInComparisonSection | ○ |
| A: Action | CTA | △要強化 |

**改善点**: CTAの訴求力強化、Offerセクションの明確化
