# SEO統合ワークフロー

## 概要

7つのSEOカテゴリを目的別に組み合わせた統合ワークフロー。タスクに応じて必要なフレームワークを選択・実行。

## タスク別ワークフロー

### 1. SEO記事作成ワークフロー

**目的**: 自然検索流入を狙った記事コンテンツ作成

**使用フレームワーク**:
- コンテンツSEO: E-E-A-T、検索意図、PREP法
- 内部SEO: 内部リンク、メタタグ
- 技術SEO: 構造化データ（Article）
- AI/AEO: 構造化回答、FAQ

**ワークフロー**:
```
Phase 1: 企画（コンテンツSEO）
├── キーワード選定
├── 検索意図分析
├── 競合上位10記事分析
└── 構成設計（見出し構造）

Phase 2: 執筆（コンテンツSEO + AI/AEO）
├── E-E-A-T要素を計画
├── PREP法で本文執筆
├── 質問形式の見出し作成
├── 50-100文字の直接回答配置
└── FAQ セクション追加

Phase 3: 最適化（内部SEO + 技術SEO）
├── メタタグ設定（title, description）
├── 内部リンク設置（3-5箇所）
├── Article Schema実装
├── FAQPage Schema実装
└── 画像最適化（alt, WebP）

Phase 4: 公開・検証
├── Search Console登録
├── 順位・CTRモニタリング
└── 定期更新（月1回）
```

---

### 2. SEO LP作成ワークフロー

**目的**: コンバージョン獲得を狙ったランディングページ

**使用フレームワーク**:
- コンテンツSEO: LP設計、PASONA、E-E-A-T
- 内部SEO: メタタグ、URL設計
- 技術SEO: Core Web Vitals、構造化データ
- AI/AEO: 構造化回答

**ワークフロー**:
```
Phase 1: 設計（コンテンツSEO）
├── ターゲットキーワード選定
├── 検索意図とCV目標設定
├── PASONA構成設計
└── 競合LP分析

Phase 2: ファーストビュー（コンテンツSEO）
├── H1（メインキーワード）
├── 価値訴求（具体的メリット）
├── 信頼バッジ（実績、評価）
├── CTAボタン（具体的アクション）
└── ヒーロー画像

Phase 3: コンテンツ（コンテンツSEO + AI/AEO）
├── Problem: 課題共感
├── Agitation: 問題深掘り
├── Solution: 解決策提示
├── Offer: 具体的提案
├── Narrowdown: 限定・緊急性
├── Action: CTA
├── E-E-A-T: 事例、監修者、データ
└── FAQ: よくある質問

Phase 4: 技術実装（技術SEO + 内部SEO）
├── Core Web Vitals最適化
├── モバイルファースト
├── 構造化データ（Product, FAQ, LocalBusiness）
├── メタタグ最適化
└── URL設計（短く、キーワード含む）

Phase 5: 公開・改善
├── A/Bテスト設計
├── ヒートマップ分析
├── CVR・直帰率モニタリング
└── 継続改善
```

### 3. 店舗ページ最適化ワークフロー

**目的**: ローカル検索での上位表示と来店促進

**使用フレームワーク**:
- ローカルSEO: GBP、NAP、レビュー
- コンテンツSEO: E-E-A-T、地域コンテンツ
- 技術SEO: LocalBusiness Schema
- 内部SEO: 店舗ページ構造

**ワークフロー**:
```
Phase 1: GBP最適化（ローカルSEO）
├── プロフィール100%入力
├── 写真10枚以上（外観、内観、商品）
├── サービス・商品登録
├── 営業時間（祝日含む）
└── 投稿週1回以上

Phase 2: 店舗ページ作成（コンテンツSEO + 内部SEO）
├── URL: /stores/[store-name]/
├── H1: 店舗名 + 地域キーワード
├── NAP情報（GBPと完全一致）
├── 地図埋め込み
├── アクセス情報詳細
└── 買取事例（地域名入り）

Phase 3: 技術実装（技術SEO）
├── LocalBusiness Schema
├── 営業時間Schema
├── 地図Schema
└── モバイル最適化

Phase 4: レビュー管理（ローカルSEO）
├── レビュー通知設定
├── 全レビュー48時間以内返信
├── ネガティブ対応プロセス
└── レビュー獲得施策

Phase 5: NAP一貫性（ローカルSEO）
├── 全プラットフォームNAP統一
├── ディレクトリ登録
└── 定期監査
```

### 4. ブランドSEO強化ワークフロー

**目的**: ブランド検索の最適化と評判管理

**使用フレームワーク**:
- 外部SEO: ブランドSEO、デジタルPR
- ローカルSEO: レビュー管理
- AI/AEO: AIプラットフォーム対応
- コンテンツSEO: E-E-A-T

**ワークフロー**:
```
Phase 1: 現状分析（外部SEO + AI/AEO）
├── ブランド検索結果確認
├── サジェスト汚染チェック
├── AI回答でのブランド表示確認
├── 競合ブランド分析
└── 口コミ・評判把握

Phase 2: オウンドメディア強化（コンテンツSEO）
├── 公式サイトE-E-A-T強化
├── 会社情報ページ充実
├── スタッフ・チーム紹介
├── 実績・事例ページ
└── ニュース・お知らせ

Phase 3: 外部露出拡大（外部SEO）
├── プレスリリース配信
├── メディア掲載獲得
├── インフルエンサー連携
└── 業界ディレクトリ登録

Phase 4: 評判管理（ローカルSEO + 外部SEO）
├── レビュー返信体制
├── ネガティブ情報対応
├── ポジティブ情報増強
└── SNS運用

Phase 5: AIプラットフォーム対応（AI/AEO）
├── ChatGPT/Perplexityでのブランド検索
├── 回答内容の確認・改善
├── 構造化データ強化
└── 信頼性シグナル強化
```

### 5. サイト全体SEO監査ワークフロー

**目的**: サイト全体のSEO健全性を評価・改善

**使用フレームワーク**:
- 技術SEO: 全項目
- 内部SEO: 全項目
- コンテンツSEO: E-E-A-T

**ワークフロー**:
```
Phase 1: 技術監査（技術SEO）
├── Core Web Vitals測定
├── モバイルフレンドリーテスト
├── 構造化データ検証
├── クロールエラー確認
└── インデックス状況

Phase 2: 内部SEO監査（内部SEO）
├── サイト構造分析
├── 内部リンク分析
├── URL設計評価
├── 重複コンテンツ確認
└── カノニカル設定確認

Phase 3: コンテンツ監査（コンテンツSEO）
├── 薄いコンテンツ特定
├── E-E-A-T評価
├── 更新日確認
└── 競合比較

Phase 4: 問題優先順位付け
├── 影響度×対応難易度マトリクス
├── クイックウィン特定
└── 中長期改善計画

Phase 5: 改善実行
├── 技術的問題修正
├── コンテンツ改善
├── 内部リンク最適化
└── 効果測定
```
