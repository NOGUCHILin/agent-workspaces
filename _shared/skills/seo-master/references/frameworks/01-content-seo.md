# コンテンツSEOフレームワーク

## 概要

コンテンツSEOは、ユーザーの検索意図に応える高品質なコンテンツを作成し、検索エンジンからの評価を高める施策。

## フレームワーク一覧

### 1. E-E-A-Tフレームワーク

Google品質評価の中核。Trust（信頼性）が最も重要。

| 要素 | 意味 | 実装方法 |
|------|------|----------|
| Experience | 体験 | 実際の事例、写真、日付、金額を掲載 |
| Expertise | 専門性 | 監修者情報、資格、経歴を明記 |
| Authoritativeness | 権威性 | 被リンク、メディア掲載、受賞歴 |
| Trust | 信頼性 | 更新日時、出典明記、デメリット記載 |

**ワークフロー**:
```
1. YMYL判定（金融・健康情報か）
2. 監修者・執筆者を決定
3. 一次情報（自社データ、体験）を収集
4. 出典・引用を準備
5. 更新日時・次回更新予定を設定
6. デメリット・注意点も正直に記載
```

### 2. ピラーページ/トピッククラスター

トピック権威性を構築する構造。

```
ピラーページ（包括的な親記事、3000字+）
├── クラスター記事1（詳細トピック）
├── クラスター記事2（詳細トピック）
├── クラスター記事3（詳細トピック）
└── クラスター記事4（詳細トピック）
```

**ワークフロー**:
```
1. コアトピックを選定
2. 関連キーワードをリサーチ
3. ピラーページの構成を設計
4. クラスター記事のテーマを決定
5. 内部リンク構造を設計
6. ピラーページから作成
7. クラスター記事を順次作成
8. 相互リンクを設定
```

### 3. 検索意図フレームワーク

ユーザーの検索意図に合わせたコンテンツ設計。

| 意図タイプ | 例 | 適切なコンテンツ |
|-----------|-----|-----------------|
| Informational | 「iPhone 買取 相場」 | ガイド記事、解説 |
| Navigational | 「AppleBuyers」 | ブランドページ |
| Commercial | 「iPhone 買取 比較」 | 比較記事、ランキング |
| Transactional | 「iPhone 売る」 | LP、査定ページ |

### 4. LP設計フレームワーク（EPIC CRO）

LP最適化の評価フレームワーク。

**ファーストビュー必須要素**:
1. h1（メインキーワード）
2. 価値訴求（具体的メリット）
3. 信頼バッジ（実績、評価）
4. CTAボタン
5. ヒーロー画像

**セクション構成**:
```
1. ヒーロー（ファーストビュー）
2. 課題提起/共感
3. 解決策（サービス説明）
4. 実績証明（事例、数字）
5. 比較（競合との差別化）
6. FAQ
7. 信頼セクション（認証、口コミ）
8. 最終CTA
```

### 5. 記事構成フレームワーク

#### PREP法
- Point（結論）
- Reason（理由）
- Example（具体例）
- Point（結論の再確認）

#### PASONAの法則
- Problem（問題提起）
- Agitation（共感）
- Solution（解決策）
- Offer（提案）
- Narrowing down（絞り込み）
- Action（行動喚起）

## コンテンツ品質基準

### 文字数目安
| タイプ | 文字数 |
|--------|--------|
| ガイド記事 | 5,000-8,000 |
| FAQ記事 | 3,000-5,000 |
| 比較記事 | 6,000-10,000 |
| ニュース | 2,000-3,000 |

### 見出し構造
```
H1（1つのみ）
├─ H2（3-5個）
│  └─ H3（各2-4個）
```

### 段落・文章
- 1段落 = 2-3行
- 1文 = 20-30文字（最大50文字）

### CTA配置
- 導入部（H1直後）
- 各セクション末尾
- 記事末尾
