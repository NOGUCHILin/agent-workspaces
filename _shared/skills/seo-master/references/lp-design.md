# SEO LP設計ガイド

## ファーストビュー設計

### 必須要素（上から順）
1. **h1タイトル** - メインキーワード含む
2. **価値訴求** - 「最大90%で買取」等の具体的メリット
3. **信頼バッジ** - 実績数、評価、認証
4. **CTAボタン** - 「無料査定」等の具体的アクション
5. **ヒーロー画像** - 縮小してファーストビュー内に収める

### よくある問題と解決策
| 問題 | 解決策 |
|------|--------|
| CTAがファーストビュー外 | 画像縮小、CTA上部配置 |
| CTA文言が弱い | 「詳しく見る」→「無料査定」 |
| 価値訴求が小さい | h1直下に大きく配置 |
| ロゴ重複 | ヘッダーのみに統一 |

## セクション構成

### 推奨順序
1. ヒーロー（ファーストビュー）
2. 症状/課題選択（インタラクティブ）
3. 買取事例（実績証明）
4. サービス説明（3ステップ等）
5. 比較表（競合・修理との比較）
6. FAQ
7. 信頼セクション（認証・口コミ）
8. 最終CTA

### 各セクションのCTA
- セクション末尾に必ずCTA配置
- 文脈に応じた文言変更
  - 買取事例後: 「あなたのiPhoneも査定」
  - FAQ後: 「まだ疑問がある？LINEで相談」

## モバイル最適化

### 基準
- 画面サイズ: 375×812px（iPhone 12-15）
- 1行文字数: 最大25文字
- タップエリア: 最小44×44px
- フォントサイズ: 本文16px以上
- 行間: 1.6-1.8

### レスポンシブ対応
- 3列グリッド → モバイルで1列
- 横スクロール表 → カード形式
- 固定CTAバナー（モバイル下部）

## 心理的要素

### スカーシティ（希少性）
- 「本日限定」「残り3枠」
- 過度な使用は逆効果

### ソーシャルプルーフ
- 「15万人が利用」
- Google口コミ評価
- 実際の買取事例写真

### 権威性
- 古物商許可番号
- 「修理専門スタッフ監修」
- 「年間10,000台査定実績」

## 構造化データ

### 必須Schema
- LocalBusiness
- Product（買取価格）
- FAQPage
- BreadcrumbList
- AggregateRating

### 実装例
```tsx
<ServiceSchema
  serviceName="iPhone買取サービス"
  serviceType="BuyingService"
  areaServed="日本"
/>
```

## パフォーマンス

### Core Web Vitals目標
- LCP: 2.5秒以下
- FID/INP: 100ms以下
- CLS: 0.1以下

### 最適化手法
- 画像: WebP/AVIF自動変換
- Lazy Loading
- Next.js Image使用
