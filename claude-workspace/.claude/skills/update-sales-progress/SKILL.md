# 進捗替えスキル（出品完了 → 販売完了）

kintoneの買取販売リストで、出品完了の商品を販売完了に更新するスキル。
Back Marketで実際の販売価格と販売日を確認し、kintoneに反映する。

## 前提条件

- Playwright MCPが有効であること
- kintone (japanconsulting.cybozu.com) にログイン済みであること
- Back Market seller portalにログイン済みであること
- Notionにアクセス可能であること

## 必要な情報

ユーザーから以下を取得：
- レコード番号（複数可）

## ワークフロー

### 1. kintoneでレコードを絞り込む

```
URL: https://japanconsulting.cybozu.com/k/11/
```

#### 絞り込み手順
1. 「絞り込む」ボタンをクリック
2. 「すべてのレコード」ドロップダウンをクリック
3. 「レコード番号」を選択
4. レコード番号を入力
5. 複数レコードの場合：
   - 2つ目の条件行で同様に「レコード番号」を選択・入力
   - 「いずれかの条件を満たす」ラジオボタンを選択
6. 「適用」をクリック（※「保存」ではない）

### 2. Notionで顧客名を確認

```
URL: https://www.notion.so/japanconsulting/ebaadbf255204fddb6098081a1715f9f?v=6bbe9978896c44c7a04ad8476491231c
```

発送予定リストでレコード番号に対応する顧客名を確認：
- SKU列でレコード番号を探す
- 該当行の「名前」列から顧客名を取得

### 3. Back Marketで販売情報を確認

```
URL: https://www.backmarket.co.jp/bo-seller/orders/all?page=1&pageSize=100&endDate={today}
```

#### 検索手順
1. 「表示するフィルターを増やす」をクリック
2. 「お客様の名前」フィールドに顧客名を入力
3. 検索結果から以下を取得：
   - 最終販売価格（税込）
   - 販売日（注文日）
4. SKUが複数ある場合は、SKUフィルターも使用して絞り込む

### 4. kintoneでレコードを更新

#### 一覧画面での編集
1. 該当レコードの「編集する」ボタンをクリック
2. 以下のフィールドを更新：
   - **進捗**: 「出品完了」→「販売完了」（ドロップダウン選択）
   - **最終販売価格（税込）**: Back Marketで確認した価格
3. 「保存」をクリック

#### 詳細画面での販売日入力
1. レコードの詳細を表示するリンクをクリック
2. 「レコードを編集する」をクリック
3. 「販売」セクションを開く
4. **販売日**: Back Marketで確認した日付（YYYY-MM-DD形式）
5. 「保存」をクリック

### 5. 次のレコードへ

全てのレコードが更新されるまで手順3-4を繰り返す。

## 更新するフィールド一覧

| フィールド名 | 入力値 | 場所 |
|-------------|--------|------|
| 進捗 | 販売完了 | 一覧画面で編集可 |
| 最終販売価格（税込） | Back Marketの販売価格 | 一覧画面で編集可 |
| 販売日 | Back Marketの注文日 | 詳細画面 > 販売セクション |
| 販売経路 | バックマーケット（既に設定済みの場合が多い） | 詳細画面 > 販売セクション |

## 注意事項

- 販売経路は「想定販売経路」と同じ値が設定されていることが多い
- 一覧画面には「販売日」フィールドが表示されないため、詳細画面で入力する
- 「適用」ボタンと「保存」ボタンを間違えないこと（保存はフィルタ条件の保存）

## Todoリスト管理

処理開始時にTodoリストを作成し、各レコードの進捗を管理：

```
- Notionで顧客名を確認: completed
- Back Marketで販売価格を確認: completed
- レコードXXXXXX更新（価格¥XX,XXX、販売日X/XX）: in_progress/completed
```
