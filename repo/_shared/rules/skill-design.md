# Skill Design Rules

Skillsを設計する際の原則。

---

## 意図を書く

「こうやってください」だけでなく「こういう意図でやってます」まで書く。

**悪い例:**
```markdown
# 使用方法
1. ファイルを読む
2. 変更を加える
3. テストを実行する
```

**良い例:**
```markdown
# 使用方法
1. ファイルを読む
   - 意図: 既存のコードスタイルを把握するため
2. 変更を加える
   - 意図: 既存パターンに合わせて一貫性を保つ
3. テストを実行する
   - 意図: 変更が既存機能を壊していないか確認
```

## なぜ意図が重要か

- AIは賢いからこそ、隠された意図を伝える必要がある
- 意図があると、想定外の状況でも適切に判断できる
- 「何を」だけでなく「なぜ」を伝えることで精度が段違いに上がる
