# setup-tomorrow-tasks

明日のフロント業務タスクを一括作成するスキル

## 概要

Notionの「タスク管理」ページで、明日のフロント業務タスクを一括作成し、担当者を設定できる状態にする。

## 手順

### 1. タスク管理ページを開く

```
https://www.notion.so/japanconsulting/16983ea47788800faaa3f9631dcd7e90
```

- ログインが必要な場合は野口サラ（noguchisara@japanconsulting.co.jp）でGoogleログイン

### 2. タスク作成トグルを開く

- ページ上部の「タスク管理」タイトル下にある「▶ タスク作成」トグルをクリック
- トグルが開くと4つのボタンが表示される：
  - 今日のフロント業務
  - **明日のフロント業務** ← これを使用
  - さらの特殊業務
  - バックオフィス業務

### 3. 明日のフロント業務ボタンをクリック

- 「明日のフロント業務」ボタンをクリック
- 約54ページのタスクが自動作成される
- 「○○ページを作成しました」と通知が表示される

### 4. 明日のみんなビューで確認

- 「今日のみんな」タブをクリック
- ドロップダウンから「明日のみんな」を選択
- 作成された全タスクが表示される

### 5. 担当者を自動設定（スクリプト使用）

Notion APIを使用して、担当者を一括設定できる。

#### スクリプトの使い方

```bash
cd claude-workspace/.claude/skills/setup-tomorrow-tasks/scripts
node assign-tasks.js --sha "担当者名" --denwa "担当者名" --mise "担当者名"
```

#### オプション

| オプション | 説明 |
|-----------|------|
| `--sha <名前>` | ネコポス担当・発送準備担当を指定 |
| `--denwa <名前>` | 電話番を指定 |
| `--mise <名前>` | 店舗担当を指定 |
| `--ura <名前>` | 裏/レ担当を指定（カンマ区切りで複数人可） |
| `--shuri <名前>` | 修理担当を指定（カンマ区切りで複数人可、雑賀晴士・高橋諒のみ） |
| `--hinshitsu <名前>` | 品質管理担当を指定（佐々木悠斗のみ） |
| `--date <日付>` | 対象日付 (YYYY-MM-DD形式、省略時は明日) |
| `--help` | ヘルプを表示 |

#### 実行例

```bash
# 明日のタスクに担当者を設定
node assign-tasks.js --sha "ナンユンティリゾーウー" --denwa "野口サラ" --mise "平山優大"

# 特定の日付のタスクに担当者を設定
node assign-tasks.js --mise "平山優大" --date "2025-01-23"

# 裏/レ担当と修理担当を設定
node assign-tasks.js --ura "細谷尚央,江口那都" --shuri "雑賀晴士,高橋諒" --date "2026-01-23"

# 品質管理担当を設定
node assign-tasks.js --hinshitsu "佐々木悠斗" --date "2026-01-23"
```

### 5b. 担当者を手動設定（UI操作）

UIで手動設定する場合：
- 各タスクの「担当者」列をクリック
- 該当する担当者を選択
- 全タスクに担当者を設定する

## タスク構造

作成されるタスクのカテゴリ：

- **サービス運営**
  - カスタマーサポート（電話、LINE、メール対応など）
- **買取**
  - 梱包キット作成、査定、査定商品開封など
- **修理**
  - 査定商品修理、販売商品修理
- **販売**
  - 出品、商品梱包、発送関連
- **店舗**
  - 店頭査定、店頭修理、店頭買取など
- **サービス管理**
  - BM在庫チェック、座席分担、買取価格変更など

### シフト表の役割とタスクの対応

#### 1. ネコポス担当・発送準備担当（シャ）
以下のタスクに設定：
- 梱包キット作成
- 発送商品伝票貼付
- 発送商品伝票貼付(バックマーケット)
- 商品伝票貼付(バックマーケット)
- 発送
- 商品梱包
- BM在庫チェック
- 梱包キット請求情報確認
- 在庫管理(梱包資材)
- 新品買取価格変更
- 新品未開封確認
- 整理整頓

#### 2. 電話番
以下のタスクに設定：
- カスタマー対応(電話)

#### 3. 店（店舗担当）
以下のタスクに設定：
- カスタマー対応（LINEとメール）
- 買取カスタマー対応(LINE)
- 買取カスタマー対応(メール)
- 買取ユーザー対応
- 買取支払い処理
- 買取促進
- 販売カスタマー対応（バックマーケット）
- 販売カスタマー対応(バックマーケット)
- 販売カスタマー対応(ムスビー)
- 結び
- 発送完了連絡ダブルチェック
- 発送完了連絡用ファイル作成
- 発送商品IMEI入力
- 発送商品進捗変更
- 店頭査定
- 店頭修理お渡し
- 店頭修理受付
- 店頭買取
- 店舗準備

#### 4. 裏/レ（査定・出品担当）- 複数人対応
シフト表で「裏」または「レ」になっている全スタッフに設定（タスクを複製して各担当者に1つずつ割り当て）：
- 査定
- 査定商品開封
- 査定商品アクティベーション
- 買取商品仕分け
- 出品(バックマーケット)
- 出品(ムスビー)
- 商品撮影(ムスビー)

#### 5. 修理担当 - 複数人対応（雑賀晴士・高橋諒のみ）
雑賀晴士・高橋諒がいる場合のみ設定（タスクを複製して各担当者に1つずつ割り当て）：
- 修理(販売商品)
- 修理(査定商品)
- 在庫管理(修理パーツ)

#### 6. 品質管理担当（佐々木悠斗のみ）
佐々木悠斗がいる場合のみ設定：
- 品質管理

### 削除対象タスク

以下のタスクは自動作成されますが、使用しないため削除します：
- BM販売価格変更
- 座席分担
- 在庫管理棚卸

削除スクリプト：
```bash
node delete-unused-tasks.js --date "2026-01-23"
```

## 注意事項

- このボタンは1日1回のみ実行する（重複作成に注意）
- 担当者設定はシフト表を参照して行う
- 作成後は「明日のみんな」ビューで確認すること
