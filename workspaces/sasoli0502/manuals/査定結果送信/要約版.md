# 本査定結果送信マニュアル（要約版）

最終更新: 2025年10月15日

---

## 全体フロー

```
1. Slack → Kintone → スプレッドシート
2. 関数コピー → 判定 → 価格調整
3. お客様に送信 → 完了処理
```

---

## 1. データ準備

### Slack
- **#本査定**チャンネルでレコード番号を確認

### Kintone
1. **「本査定結果送信用」**テンプレートを選択
2. レコード番号で絞り込み
3. **「ファイルに書き出す」**でダウンロード

### スプレッドシート
1. **「本査定ファイル」**を開く
2. ファイル → インポート → **「現在のシートに追加する」**
3. インポートされた**フィールド名行を削除**

---

## 2. 関数適用

1. **直前の行のK～R列**をコピー
2. 新規追加行の**K～R列**にペースト
3. 手動入力（N・O・P列）が残っていたら削除

**列の意味:**
- K: 実販売売上、L: 手数料引き粗利、M: 利益率
- N: 判定（手動）、O: 新高額（手動）、P: 新特急（手動）
- Q: 新粗利、R: 新利益率

---

## 3. 判定（N列に○か×を入力）

### F列「ネットワーク利用制限」で分岐

#### ○の場合
1. G列が「バックマーケット以外」→ **判定○**
2. G列が「バックマーケット」→ L列・M列を確認
   - 最低基準を満たす → **判定○**
   - 満たさない → **判定×**

**最低基準:**
| 販売価格（K列） | 最低粗利（L列） | 最低利益率（M列） |
|---------------|---------------|-----------------|
| 0～2万 | 4,000円 | 20% |
| 2～3万 | 5,000円 | 10% |
| 3～4.5万 | 6,000円 | 15% |
| 4.5～6万 | 7,000円 | 15% |
| 6～9万 | 8,000円 | 11% |
| 9～11万 | 9,000円 | 11% |
| 11～15万 | 10,000円 | 10% |
| 15万～ | 10,000円 | 10% |

#### △の場合
1. **O列・P列に減額計算式を入力:**
   - O列: `=H2-MIN(20000,MAX(20,ROUND(H2*IFS(H2<10000,0.10,H2<50000,0.07,H2<100000,0.06,H2<=200000,0.10),0))))`
   - P列: `=I2-MIN(20000,MAX(20,ROUND(I2*IFS(I2<10000,0.10,I2<50000,0.07,I2<100000,0.06,I2<=200000,0.10),0))))`
   - （H2・I2は対応する行番号に変更）

2. 減額後にQ列・R列を確認
   - G列が「バックマーケット以外」→ **判定○**
   - G列が「バックマーケット」→ Q列・R列が最低基準を満たすか確認
     - 満たす → **判定○**
     - 満たさない → **判定×**

#### ×・空白の場合
- **判定○**（価格が事前に下がっているため）

---

## 4. 判定×の価格調整

### 調整手順
1. **P列（新特急）から調整**
   - Q列・R列を見ながら最低基準を満たす価格に
   - 低単価（5万円以下）: 利益率30%目指す
   - 高単価（5万円以上）: 最低利益率+5%程度

2. **O列（新高額）を決定**
   - 新特急との差をつける
   - お客様が高額を選びたくなるように
   - ただし新高額も最低基準を満たすこと

**重要:** 極端な減額は避ける

---

## 5. 送信

### 判定○の場合
- Kintoneの情報をそのまま使用
- 写真がなければSlackで本査定スタッフに依頼

### 判定×の場合

#### Kintoneで価格変更
1. **「メッセージ生成」→「本査定完了メッセージ」**
   - 特急買取価格 → 新特急（P列）
   - 高額買取価格 → 新高額（O列）

2. **下にスクロール→「価格情報」**
   - 最高・最低提示価格（特急）→ 新特急
   - 最高・最低提示価格（高額）→ 新高額

3. **「査定結果は以下のようになりました」を削除して置き換え:**
   ```
   大変申し訳ございません。
   新機種発売に伴い、買取価格の相場が大きく変動しています。
   以下の買取金額となります。
   ```

4. 送信

---

## 6. 完了処理

1. **スプレッドシート:** A列（レコード番号）を灰色に変更
   - 色ボタン → 一番上の行の右から4番目
2. **Slack:** #本査定チャンネルでスタンプ

---

## よくある質問

**Q: 判定に迷ったら？**
A: 保守的に判定○でOK

**Q: △の減額計算が複雑？**
A: O列・P列に計算式を入力すれば自動計算される

**Q: 相場変動の文章は？**
A: 判定×の場合のみ
