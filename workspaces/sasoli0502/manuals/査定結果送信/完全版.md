# 本査定結果送信マニュアル（完全版）

最終更新: 2025年10月15日

---

## 全体フロー

```
1. Slackで対象レコード番号を確認
   ↓
2. Kintoneでデータを絞り込み・書き出し
   ↓
3. スプレッドシートにインポート
   ↓
4. インポート後の処理（フィールド名削除、関数適用）
   ↓
5. 判定と価格調整
   ↓
6. お客様に本査定結果を送信
   ↓
7. 送信完了後の処理
```

---

## 1. Slackで対象レコード番号を確認

1. **Slackアプリ**を開く
2. **#本査定**チャンネルを開く
3. スタッフが報告した**レコード番号**を確認
   - 本査定結果送信ができる段階まで進んだレコード

---

## 2. Kintoneでデータを絞り込み・書き出し

### 2-1. フィールド並び替えテンプレートを選択
1. **kintone**アプリを開く
2. **「本査定結果送信用」**テンプレートを選択

### 2-2. レコードを絞り込み
1. **絞り込み機能**を使用
2. Slackの#本査定チャンネルにまとめられた**レコード番号**を入力
3. 対象レコードが表示される

### 2-3. ファイルに書き出し
1. **「ファイルに書き出す」**ボタンを押す
2. PCにダウンロードする

---

## 3. スプレッドシートにインポート

### 3-1. スプレッドシートを開く
- **「本査定ファイル」**スプレッドシートを開く

### 3-2. インポート操作
1. 左上の**「ファイル」**ボタン → **「インポート」**を選択
2. **「アップロード」** → **「参照」**ボタンを押す
3. **「ダウンロード」**フォルダを選択
4. Kintoneから書き出したファイルを選択 → **「開く」**
5. インポート方法で**「現在のシートに追加する」**を選択
6. インポート完了

---

## 4. インポート後の処理

### 4-1. フィールド名行を削除
- インポートすると一番下の空白行に**フィールド名**が出現
- その下に数値・文字データが並ぶ
- **フィールド名の行を削除する**

### 4-2. インポート後のフィールド構成（A～J列）

| 列 | フィールド名 |
|----|-------------|
| A列 | レコード番号 |
| B列 | ユーザー氏名 |
| C列 | 機種 |
| D列 | 容量 |
| E列 | ランク |
| F列 | ネットワーク利用制限 |
| G列 | 想定販売経路 |
| H列 | 最高提示価格（高額） |
| I列 | 最高提示価格（特急） |
| J列 | 想定販売価格 |

**ここまでが自動インポート**

### 4-3. K～R列に関数を適用

新しく追加された行の**J列より右は空白**なので、関数をコピーする。

#### 操作手順:
1. **直前の行**（新規追加行の1つ上の行）の**K～R列**を選択してコピー
2. 新規追加された行の**K～R列全体**を選択
3. ペースト

#### 注意点:
- もし直前の行に**手動入力（N列判定、O列新高額、P列新特급）**が入っていた場合、新規行ではそれを削除（関数だけ残す）
- ドラッグではなく**コピー&ペースト**で一括処理すること

#### K～R列のフィールド一覧:

| 列 | フィールド名 | 種類 |
|----|-------------|------|
| K列 | 実販売売上 | 関数 |
| L列 | 手数料引き粗利 | 関数 |
| M列 | 利益率 | 関数 |
| N列 | 判定 | **手動入力** |
| O列 | 新高額 | **手動入力** |
| P列 | 新特急 | **手動入力** |
| Q列 | 新粗利 | 関数 |
| R列 | 新利益率 | 関数 |

---

## 5. 判定と価格調整

### 5-1. F列「ネットワーク利用制限」による判定フロー

#### ケース1: F列が「○」の場合

1. **G列「想定販売経路」**を確認
   - **バックマーケット以外** → **無条件で判定「○」**（N列に「○」と入力）
   - **バックマーケット** → 次のステップへ

2. **バックマーケットの場合、L列・M列を確認:**

   **最低基準（実験ルール）:**
   | 販売価格（K列） | 最低粗利（L列） | 最低利益率（M列） |
   |---------------|---------------|-----------------|
   | 0～20,000円 | 4,000円 | 20% |
   | 20,000～30,000円 | 5,000円 | 10% |
   | 30,000～45,000円 | 6,000円 | 15% |
   | 45,000～60,000円 | 7,000円 | 15% |
   | 60,000～90,000円 | 8,000円 | 11% |
   | 90,000～110,000円 | 9,000円 | 11% |
   | 110,000～150,000円 | 10,000円 | 10% |
   | 150,000円～ | 10,000円 | 10% |

   - K列の販売価格帯に対応する最低基準と比較
   - L列（粗利）≧ 最低粗利 **かつ** M列（利益率）≧ 最低利益率
     - **満たす** → N列に「○」と入力
     - **満たさない** → N列に「×」と入力 → **O列・P列に新高額・新特急を入力**

---

#### ケース2: F列が「△」の場合

**△の場合は価格を減額してから判定する**

##### ステップ1: 減額計算

**減額ルール:**
| 価格帯 | 減額率 | 備考 |
|--------|--------|------|
| ～9,999円 | 10% | 最低20円 |
| 10,000～49,999円 | 7% | - |
| 50,000～99,999円 | 6% | - |
| 100,000～200,000円 | 10% | 上限20,000円 |

**減額関数:**
```
=MIN(
  20000,
  MAX(
    20,
    ROUND(
      [対象価格] *
      IFS(
        [対象価格]<10000, 0.10,
        [対象価格]<50000, 0.07,
        [対象価格]<100000, 0.06,
        [対象価格]<=200000, 0.10
      ),
    0)
  )
)
```

**計算方法:**
1. **O列「新高額」に計算式を入力:**
   - `=H列 - 減額関数`
   - 例: H列が2行目の場合 `=H2-MIN(20000,MAX(20,ROUND(H2*IFS(H2<10000,0.10,H2<50000,0.07,H2<100000,0.06,H2<=200000,0.10),0)))`

2. **P列「新特急」に計算式を入力:**
   - `=I列 - 減額関数`
   - 例: I列が2行目の場合 `=I2-MIN(20000,MAX(20,ROUND(I2*IFS(I2<10000,0.10,I2<50000,0.07,I2<100000,0.06,I2<=200000,0.10),0)))`

**注意:** H列・I列の値は変更しない。O列・P列に計算式を入力すること。

**計算例:**
- H列が50,000円の場合
  - 減額率7% → 減額額3,500円
  - O列に `=H2-MIN(20000,MAX(20,ROUND(H2*IFS(H2<10000,0.10,H2<50000,0.07,H2<100000,0.06,H2<=200000,0.10),0)))` と入力
  - 結果: 46,500円

##### ステップ2: 減額後に判定

O列・P列に減額後の価格を入力すると、Q列「新粗利」とR列「新利益率」が自動計算される。

1. **G列「想定販売経路」**を確認
   - **バックマーケット以外** → **N列に「○」**
   - **バックマーケット** → 次のステップへ

2. **バックマーケットの場合、Q列・R列を確認:**
   - K列の販売価格帯に対応する最低基準と比較
   - Q列（新粗利）≧ 最低粗利 **かつ** R列（新利益率）≧ 最低利益率
     - **満たす** → N列に「○」
     - **満たさない** → N列に「×」→ O列・P列をさらに調整

---

#### ケース3: F列が「×」または空白の場合

- **無条件で判定「○」**（N列に「○」と入力）
- 事前に価格が大幅に下がっているため
- G列も「バックマーケット以外」になる

---

### 5-2. 判定「×」の場合の価格調整方針

判定が「×」の場合、O列・P列に調整後の価格を入力する。

**調整の仕組み:**
- P列「新特急」に数字を入れると、Q列「新粗利」とR列「新利益率」が自動計算される
- まず新特急を調整して、新粗利・新利益率を確認
- その後、新高額を決定

#### 調整の目安:

**新特急（P列）:**
- **低単価（販売価格5万円以下）**: 積極的に利益率30%を目指す
- **高単価（販売価格5万円以上）**: 慎重に、最低利益率+5%程度

**新高額（O列）:**
- 新特急との差をつけて、お客様が高額買取を選びたくなるように設定
- ただし、新高額も最低粗利・最低利益率を満たす必要がある
- ここは抽象的で難しいが、慎重に決める

**重要:** お客様の不信感につながる極端な減額は避ける

---

## 6. お客様に本査定結果を送信

### 6-1. 判定「○」の場合

1. Kintoneの情報をそのまま使用して送信
2. 本査定を行ったスタッフが撮影した**傷の写真・不良の写真**がKintoneに入っている
3. それを使って送信

#### 写真がない場合:
- Slackの**#本査定チャンネル**で該当レコードの**スレッドに返信**
- 本査定スタッフに写真の追加を依頼

---

### 6-2. 判定「×」の場合（価格変更あり）

#### ステップ1: 本査定完了メッセージの設定

**すでに設定している人:**
- Kintoneの**「メッセージ生成」** → **「本査定完了メッセージ」**を開く
- **特急買取価格**を「新特急（P列の値）」に変更
- **高額買取価格**を「新高額（O列の値）」に変更

**設定していない人:**
- 自分で本査定完了メッセージを設定
- 価格を「新特急」「新高額」に変更

#### ステップ2: Kintoneの価格情報を変更

1. Kintoneを下にスクロールして**「価格情報」**セクションを開く
2. 以下のフィールドを変更:
   - **最高提示価格（特急）** → 新特急（P列の値）
   - **最低提示価格（特急）** → 新特急（P列の値）
   - **最高提示価格（高額）** → 新高額（O列の値）
   - **最低提示価格（高額）** → 新高額（O列の値）

**注意:** 本査定完了後は最高提示価格と最低提示価格が同じ数字になります。

#### ステップ3: 相場変動の文章を追加

本査定完了メッセージの中に「査定結果は以下のようになりました」という文があるので、これを削除して以下のテンプレートに置き換える:

```
大変申し訳ございません。
新機種発売に伴い、買取価格の相場が大きく変動しています。
以下の買取金額となります。
```

#### ステップ4: 送信

設定が完了したら、お客様に本査定結果を送信。

## 7. 送信完了後の処理

### 7-1. スプレッドシートで色変更

1. 送信完了したレコードの**A列（レコード番号）**を選択
2. 色変更ボタンを押す
3. 一番上の行の**右から4番目の灰色**を選択
4. セルの色が灰色に変わる

### 7-2. Slackでスタンプ

1. Slackの**#本査定チャンネル**を開く
2. 該当レコードの報告に**スタンプ**をつける
3. 「送信完了」の印

## 8. よくある質問

### Q1: 直前の行に手動入力が残っている場合は？
A: 新規行にペースト後、N列・O列・P列の手動入力部分を削除してください。関数のみ残します。

### Q2: ネットワーク利用制限が「△」で計算が複雑な場合は？
A: まず減額額を計算し、H列-減額額をO列に、I列-減額額をP列に入力。その後、Q列・R列を確認して判定してください。

### Q3: 判定に迷った場合は？
A: 保守的に判定「○」でOK。無理に価格を下げて返送になるより、確実に買取する方が重要です。

### Q4: 相場変動の文章は必ず入れる？
A: 判定「×」（価格変更あり）の場合のみ追加してください。判定「○」の場合は不要です。
