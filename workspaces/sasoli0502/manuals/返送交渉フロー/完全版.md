## 返送交渉とは

### 背景
- お客様が本査定結果に納得できず「返送してほしい」と連絡してくる
- そのまま返送すると買取機会を失う
- 価格を調整して交渉することで買取成立を目指す

### 目的
1. **顧客満足度の向上**: 柔軟な対応でお客様の期待に応える
2. **売上の確保**: 返送を避けて買取を成立させる
3. **利益の確保**: 最低利益基準を守りながら交渉

### 対応スピードの重要性
- お客様は「返送してください」と送った時点で不満を持っている
- 当日中に返信がないとさらに不満が増幅する
- **可能な限り当日中に1回目の交渉メッセージを送る**

・売上総利益（粗利）を確認
・利益率を確認

  ├─ YES → 交渉開始
  └─ NO  → 交渉不成立（そのまま返送手続き）
  ↓
【1回目の交渉】

  - 不具合の減額取り消し（カメラ・バッテリー・画面）

## 価格決定の考え方

### 基本原則

#### 1. 厳密なパターン判定はない
- **お客様の文脈次第で柔軟に対応**
- 減額取り消しを理由にするか、別の理由にするかは自由
- お客様が納得すればOK

#### 2. 文脈を読む
お客様のメッセージから:
- **口調**: 柔らかい / 強い / 冷静
- **要求の強さ**: 「できれば…」/ 「絶対に…」
- **具体性**: 希望金額を提示 / 漠然と不満を表明

例:
- 「もう少し上がれば嬉しいです」→ ちょっとだけ価格UP
- 「この金額では絶対に無理です」→ 大きく価格UP + 理由付け

#### 3. 価格の上げ方
- **段階的に上げる**: 最大値まで一気に上げない
- **極端な値上げは避ける**: 不信感につながる
  - 例: ❌ 1万円 → 5万円（「なぜ最初から高く提示しなかった？」）
- **上げ幅の目安**: 最大値の「半分程度」から開始（ただし柔軟に）
- **利益率とのバランス**: 最低利益基準を守る

### スプレッドシートでの確認

1. **該当レコードを検索**
   - レコード番号で検索
   - セル色を青色に変更（交渉中を可視化）

2. **確認項目**
   - 機種・容量
   - 現在の買取価格（特急・高額）
   - 販売先
   - 売上総利益（粗利）
   - 利益率
   - ネットワーク利用制限
   - 不具合の有無

3. **計算**
   - スプレッドシートの関数を使って計算
   - 最低利益基準を満たすか確認
   - 最大値を把握

## メッセージ作成ガイド

### 1回目（テンプレート）

#### 基本テンプレート
```
お世話になっております。
ご連絡ありがとうございます。

社内で協議した結果、以下の金額で対応可能となりました。
いかがでしょうか。

【買取価格】
特急買取: ○○円
高額買取: ○○円

ご検討のほどよろしくお願いいたします。
```

#### ポイント
- 簡潔・丁寧に
- 感情的な表現は入れない
- 協議した結果という体裁を保つ

---

### 2回目以降（柔軟対応）

#### 基本方針
- **文脈を読む**: お客様のトーン・要求に合わせる
- **感情を出す**: 「上司に相談したところ…」など
- **段階的に**: いきなり最大値を出さない

#### パターン1: さらに価格を上げる場合
```
お返事ありがとうございます。

上司に再度相談いたしまして、
以下の金額まで対応可能となりました。

【買取価格】
特急買取: ○○円
高額買取: ○○円

何卒ご検討いただけますと幸いです。
```

#### パターン2: 希望金額を聞く場合（2回目で断られた時）
```
承知いたしました。
返送のお手続きを進めさせていただきます。

ご参考までに、ご希望の買取金額を
ご教示いただけますと幸いでございます。
```

**重要**: 返送手続きを進めつつ希望金額を聞く
- お客様が金額を提示 → 対応可能なら返送を取り消し
- 対応不可能 → そのまま返送
- 手続き完了後 → 対応不可

## 理由付けのバリエーション

### パターンA: 不具合の減額取り消し
対象不具合（交換不要レベル）:
- **カメラの曇り**: 人によって視認できる/できないレベル
- **非純正バッテリー**: 動作に問題なし
- **非純正画面**: 品質に問題なし

メッセージ例:
```
再度確認したところ、○○の不具合は
軽微なレベルと判断いたしました。

減額を取り消し、以下の金額で対応いたします。
```

**注意**: 外装ランクの変更は行わない（信頼性の問題）

### パターンB: 相場変動
```
新機種発売に伴い、買取価格の相場が変動しています。
再度協議した結果、以下の金額で対応可能となりました。
```

### パターンC: 社内協議
```
社内で協議した結果、以下の金額で対応可能となりました。
```

## スプレッドシート管理

### 交渉中のレコード管理
- **セル色を青色に変更**: 誰が見ても「交渉中」と分かる
- **交渉終了時**: セル色を元に戻す
- 詳細はスプレッドシートを参照

## よくある質問

### Q1: 1回目でいくら上げるべき？
**A**: 明確な基準はない。お客様の文脈を読んで柔軟に判断。目安は最大値の「半分程度」だが、状況次第で変える。

### Q2: 不具合の減額取り消しは必ずやる？
**A**: 必須ではない。お客様が納得すればどんな理由でもOK。柔軟に判断。

### Q3: 外装ランクを上げての交渉はできる？
**A**: できない。信頼性の問題につながるため、外装ランクは変更しない。

### Q4: 希望金額を聞くタイミングは？
**A**: 2回目で断られたら。返送手続きを並行して進める。

### Q5: 最低利益基準を下回ってもいい？
**A**: 基本的にNG。特別な理由がある場合は上司に相談。

### Q6: 交渉は何往復まで？
**A**: 基本的に2回程度。それ以降は希望金額を聞いて返送手続き。

### Q7: 返送手続き中に希望金額が来たら？
**A**: 対応可能なら返送キャンセル。ただし手続き完了後は無理。

### Q8: お客様が感情的になっている場合は？
**A**: 冷静に対応。感情に流されず、最低利益基準を守る。難しい場合は上司に引き継ぐ。
