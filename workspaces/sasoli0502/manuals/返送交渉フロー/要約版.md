# 返送交渉マニュアル（要約版）

最終更新: 2025年10月15日

---

## 全体フロー

```
1. お客様から返送希望 → スプレッドシートで確認
2. 文脈を読んで価格決定 → 1回目メッセージ送信
3. お客様の反応確認 → 2回目以降は柔軟対応
4. 交渉不成立 → 希望金額を聞いて返送手続き
```

---

## 交渉フロー図

```
お客様「返送してください」
  ↓
スプレッドシートで確認（セル青色に）
  ↓
価格を上げられる？
  ├─ YES → 交渉開始
  └─ NO  → 交渉不成立（そのまま返送手続き）
  ↓
【1回目】
文脈を読んで価格決定
・柔らかい口調 → 少し上げる
・「少し上がりませんか」→ 少し上げる（買取価格に応じて）
・強い要求 → しっかり上げる
・重要: 上げ切れる上限まで上げ切らない
・理由付けは柔軟に（選択肢が多いほど説得力UP）
  - 不具合の減額取り消し
  - 相場変動
  - 社内協議の結果
  ↓
テンプレートで送信（当日中）
  ↓
お客様の反応は？
  ├─ OK → 交渉成立
  ├─ もう少し → 【2回目】へ
  └─ 無理です → 希望金額を聞く＋返送手続き（並行）
  ↓
【2回目以降】
理由付けの柔軟さを強化:
・「上司に相談したところ…」
・感情的な表現（お客様のご希望に添えず申し訳ございません、等）
・価格を再提示
  ↓
お客様の反応は？
  ├─ OK → 交渉成立
  └─ 無理です → 希望金額を聞く＋返送手続き（並行）
  ↓
返送手続き中に希望金額が来た
  ↓
対応可能？
  ├─ YES → 返送キャンセル → 交渉成立
  └─ NO  → そのまま返送
```

---

## 重要ポイント

### 1. パターン判定は厳密ではない
- **お客様の文脈次第で柔軟に対応**
- 例: 少し不満そう → ちょっとだけ価格UP
- 例: 強い要求 → 大きく価格UP + 理由付け
- 理由付けの選択肢:
  - 不具合の減額取り消し（カメラ・バッテリー・画面）
  - 相場変動
  - 社内協議の結果
  - 上司との相談の結果
  - 再度ご確認させていただいたところ

### 2. スピード最優先
- **当日中に1回目のメッセージを必ず送る**
- **可能であれば即座に送る（チャットを開いた人が即座にメッセージを送る）**
- 返信が遅れるとお客様の不満が増幅

### 3. 価格の上げ方
- 最大値まで一気に上げない（段階的に）
- ただし極端な値上げは避ける（不信感につながる）
- 例: ❌ 1万円 → 5万円（お客様が疑問に思う）
- 利益率と文脈のバランスを見る

### 4. 希望金額を聞くタイミング
- **2回目で断られたら**
- 返送手続きを並行して進める
- 対応可能な金額なら返送をキャンセル

### 5. スプレッドシート管理
- 交渉中のレコードはセル色を青色に
- 詳細はスプレッドシートを参照

## よくある質問

**Q: どのくらい価格を上げればいい？**
A: お客様の文脈次第。明確な基準はない。

**Q: 不具合の減額取り消しは必ずやる？**
A: お客様が納得すればどんな理由でもOK。柔軟に判断。

**Q: 外装ランクは変更できる？**
A: できない。信頼性の問題につながる。

**Q: 交渉は何往復まで？**
A: 基本的に2回程度。それ以降は希望金額を聞いて返送手続き。

**Q: 返送手続き中に希望金額が来たら？**
A: 対応可能なら返送キャンセル。ただし手続き完了後は無理。
