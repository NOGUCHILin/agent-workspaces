## 労務管理上の重要事項（社労士指導に基づく）

### 退職・解雇に関する基本原則

#### 期間の定めがない従業員（正社員等）
- **いきなり解雇はほぼ認められない**
- 解雇には**指導や懲戒の蓄積**と**改善させようとした会社の努力**が必要
- 原則として**話し合いによる合意（退職勧奨）**で解決する

**対応フロー（例：無断遅刻・早退・欠勤が頻発する場合）**
1. **指導書**を発行し、改善を求める（必要に応じて誓約書も取得）
   - この時点で懲戒事由に該当すれば、程度により譴責や減給も検討
2. 改善が見られない場合、**業務指示書**や**最終警告書**を積み上げる
3. それでも改善がない場合、契約終了を検討

#### 期間の定めがある従業員（契約社員等）
- 契約期間満了により、期間更新をしないことで退職となる
- **最後の契約期間開始前に雇用契約書を渡す**（契約満了1か月以上前通知）
- 最後の契約期間の雇用契約書には「**本契約は更新しない**」と明記

**⚠️ 重要な例外**
- **3回以上契約更新**されている労働者
- **1年を超えて継続雇用**されている労働者

これらの場合、契約期間満了であっても解雇乱用になりうるため、期間の定めがない従業員と同様に**指導・懲戒の蓄積が必要**

### 時給減額の考え方

減額の正当な理由：
- 勤務状況が不安定な人に任せられる業務は限られる
- 能力や任せられる仕事内容に応じて賃金を決めないと、他の従業員から不満が出る
- **能力、仕事内容、勤怠状況等を考慮**して判断する

⚠️ **避けるべき理由**：「盗んだかもしれない」等、証拠がない状況での減給

### 必要書類とその使用目的

会社には以下の書式が用意されています（`files/`ディレクトリ参照）：

1. **書式42：指導書**
   - 問題行動の初期段階で使用
   - 無断欠勤・遅刻、誠実労働義務違反等の指摘
   - 改善を求め、次回同様の行為があれば懲戒処分を行う旨を警告

2. **書式43：業務指示書**
   - 具体的な業務内容と期限を明示
   - 進捗確認の面談日時も設定
   - 業務遂行能力の改善プロセスを記録

3. **書式44：最終警告書**
   - 複数回の指導・懲戒処分を経ても改善が見られない場合
   - これまでの指導・面談・処分の履歴を記載
   - 労働契約終了もありうる旨の最終通告

### 文書管理の重要性

- すべての指導・懲戒は**文書で記録**し、保管すること
- 面談の日時・内容も記録に残すこと
- これらの記録が、万が一の労働紛争時に会社を守る証拠となる

## 注意事項

- **個人情報保護**: 個人データは必ず`.gitignore`に登録し、Git管理外とすること
- **機密情報**: パスワードやAPIキーなどは環境変数または別管理
- **バックアップ**: 重要データは定期的にバックアップを取ること

## データファイルの扱い

以下のパターンのファイルは自動的にGit管理外となります：
- `data/`配下のすべてのファイル
- `files/`配下のすべてのファイル
- `*.xlsx`, `*.csv`（個人情報を含む可能性があるため）
