# Back Market 発送完了連絡スキル

Back Marketで販売した注文に対し、ヤマトB2クラウドで発行済みの送り状から追跡番号を取得し、Back Marketで発送完了処理を行う。

## 前提条件

- Playwright MCPが有効であること
- B2クラウドにログイン済みであること
- Back Market seller portalにログイン済みであること

## ワークフロー

### 1. Back Marketで発送予定注文を確認

```
URL: https://www.backmarket.co.jp/bo-seller/orders/all?page=1&pageSize=100&endDate={today}&state=MERCHANT_NEED_TO_SEND
```

「発送予定」ステータスの注文一覧から以下を取得：
- 注文番号
- SKU（末尾の's'などの文字は検索時に除外）
- お客様名

### 2. B2クラウドで追跡番号を検索

```
URL: https://newb2web.kuronekoyamato.co.jp/issue_search.html
```

#### 検索手順
1. 「詳細検索オプション」を開く
2. 「品名1」フィールドにSKU番号を入力（末尾の文字's'等は除外）
3. 「検索」をクリック

#### 検索結果の確認
- お届け先名がBack Marketの注文と一致することを確認
- 2件の結果が表示される（同じSKUで2つの送り状番号）

#### 追跡番号の特定
1. 検索結果を全選択
2. 「荷物のお問い合わせ」をクリック
3. ポップアップで「荷物受付」ステータスの追跡番号を確認
   - パターン: `4694-3950-XXXX` 形式
   - 「荷物受付」ステータスの方を使用
4. ポップアップを閉じる

### 3. Back Marketで発送完了処理

1. Back Marketタブに切り替え
2. 該当注文の「発送する」ボタンをクリック
3. 配送情報ダイアログで：
   - 配送業者: Yamato（デフォルト）
   - 追跡番号: B2クラウドで確認した番号を入力
4. 「アイテムを発送します」をクリック
5. 完了メッセージを確認

### 4. 次の注文へ

全ての「発送予定」注文が処理されるまで手順2-3を繰り返す。

## 名前照合ルール

- B2クラウドのお届け先名とBack Marketの顧客名が一致することを確認
- 不一致の場合は処理を停止してユーザーに確認

## エラー処理

| 状況 | 対応 |
|------|------|
| SKUで検索結果なし | ユーザーに報告、スキップ |
| 名前不一致 | ユーザーに報告、確認を求める |
| 荷物受付ステータスなし | ユーザーに報告、スキップ |

## Todoリスト管理

処理開始時にTodoリストを作成し、各注文の進捗を管理：

```
- 注文XXXXXX (SKU: YYYYYY) - 顧客名: pending/in_progress/completed
```

## 関連スキル

- `yamato-login`: ヤマトB2クラウドへのログイン
