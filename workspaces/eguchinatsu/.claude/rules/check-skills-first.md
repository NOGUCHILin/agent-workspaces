# スキル確認ルール

## タスク実行前のスキル確認（必須）

ユーザーから「やって」「して」「作って」「開いて」などのタスク依頼を受けた場合：

1. **まず関連するスキルがあるか確認する**
2. 該当スキルがあれば、そのスキルを使用する
3. 該当スキルがなければ、通常の方法で実行する

## スキルとキーワードの対応

| キーワード | 使用すべきスキル |
|-----------|----------------|
| 調べて、検索して、最新情報 | `/search` |
| 画像作って、バナー作って、イラスト | `/generate-image` |
| 音声作って、ナレーション、歌 | `/generate-audio` |
| 動画作って、映像作成 | `/generate-video` |
| 仕様書作って、新しいタスク、要件定義 | `/create-spec` |
| セットアップ、MCP設定、連携 | `/setup-mcp` |
| 他のプロジェクト、一貫性チェック | `/reference-parent` |
| キーバインド、ショートカット | `/keybindings-help` |
| LINE開いて、LINEチャット | `/open-line` |
| 仮査定催促、仮査定後催促 | `/karisatei-saisoku` |
| 本査定催促、本査定後催促 | `/honsatei-saisoku` |
| 梱包キット催促、キット催促 | `/konpokit-saisoku` |
| お疲れ様、終わり、ブラウザ閉じて | `/close-browser` |
| おはようございます、おはよう | `/good-morning` |

## 注意

- スキルの存在確認を省略しない
- 不明な場合はユーザーに確認する
