# Building Effective Agents

出典: [Anthropic Engineering](https://www.anthropic.com/engineering/building-effective-agents)

---

## 主要な知見

> 「最も成功した実装は複雑なフレームワークではなく、シンプルで合成可能なパターンを使用」

---

## アーキテクチャの区別

| 種類 | 説明 |
|------|------|
| **ワークフロー** | 事前定義されたコードパスでLLMとツールを制御 |
| **エージェント** | LLMが自身のプロセスとツール使用を動的に指示 |

---

## 実装パターン（段階的複雑性）

| # | パターン | 説明 |
|---|---------|------|
| 1 | 拡張LLM | 検索・ツール・メモリで強化 |
| 2 | プロンプトチェーニング | 連続LLM呼び出しで固定タスク分解 |
| 3 | ルーティング | 入力分類→専門化処理経路へ |
| 4 | 並列化 | 複数インスタンス同時処理→集約 |
| 5 | オーケストレータ-ワーカー | 中央LLMが動的サブタスク配分 |
| 6 | 評価者-最適化者 | フィードバックループで反復改善 |
| 7 | 自律エージェント | 環境フィードバックに基づく長期運用 |

---

## コア原則

| 原則 | 説明 |
|------|------|
| **Simplicity** | 設計をシンプルに維持 |
| **Transparency** | 計画ステップを明示 |
| **ACI設計** | Agent-Computer Interfaceの綿密な設計 |

---

## ツール最適化

> 「SWE-benchエージェント構築時、プロンプトよりツール最適化に多くの時間を費やした」

例: 相対パス → 絶対パス要求に変更で信頼性が飛躍的向上
