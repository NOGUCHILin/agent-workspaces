# Agent Skills

出典: [Anthropic Engineering](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills)

---

## 概要

Agent Skillsは、Claudeが特殊タスクを処理するためのモジュラーシステム。
カスタムエージェント設計なしで、汎用エージェントをドメイン特化ツールに変換。

---

## コアコンセプト

> 「整理されたフォルダ（指示、スクリプト、リソース）をエージェントが動的に発見・読み込み」

**Progressive Disclosure**: タスクに関連する場合のみ必要なコンテキストを読み込み。

---

## Skillの構造

`SKILL.md`ファイルを含むディレクトリ。3段階の開示レベル:

| レベル | 内容 | 読み込みタイミング |
|-------|------|-------------------|
| 1 | YAMLフロントマター（name, description） | 起動時 |
| 2 | SKILL.md全内容 | 関連性判断時 |
| 3+ | reference.md, forms.md等 | 必要時のみ |

---

## 実例: PDF Skill

PDF直接操作（LMの制限）の代わりに、フォームフィールドを抽出するPythonスクリプトをバンドル。
→ コード実行が指示ベースガイダンスを補完。

## Skill設計・組織化

### 数と分割

| 項目 | ガイドライン |
|------|-------------|
| **上限** | 明示なし（100個以上でも動作） |
| **分割基準** | 1 Skill = 1 目的 |
| **サイズ** | 500行超えたら参照ドキュメントに分割 |

### 命名規則

- `動詞-名詞`形式（例: `create-worktree`）
- 小文字・ハイフンのみ、64文字以内
- descriptionは3人称で「何ができるか」「いつ使うか」を明記

### 重複への対処

descriptionで使い分けを明記:
```
create-worktree: 「既存プロジェクトに新ブランチ追加時」
setup-project: 「新規プロジェクト作成時」
```

## 開発ガイドライン

| 項目 | 説明 |
|------|------|
| **評価から開始** | 代表的タスクテストで能力ギャップを特定 |
| **スケール構造** | 大きなSKILL.mdは参照ドキュメントに分割 |
| **Claudeの視点** | 実際の使用パターンを監視、説明を反復改善 |
| **協調的反復** | 成功アプローチを再利用可能コンテキストに |
